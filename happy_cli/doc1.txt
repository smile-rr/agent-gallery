## AWS 服务速查表

**计算**

| 类别          | 服务                             | 描述                                                         |
| ------------- | -------------------------------- | ------------------------------------------------------------ |
| 实例 (虚拟机) | EC2                              | 提供云中安全的、可调整大小的计算能力。它使 Web 级别的云计算更容易实现。 |
| 实例 (虚拟机) | EC2 Spot                         | 为工作负载运行，价格低至 90%。                               |
| 实例 (虚拟机) | EC2 AutoScaling                  | 根据需求自动添加或删除计算能力，以满足需求变化。             |
| 实例 (虚拟机) | Lightsail                        | 旨在以最简单的方式启动和管理虚拟私有服务器，并提供易于使用的云平台，涵盖构建应用程序或网站所需的一切。 |
| 实例 (虚拟机) | Batch                            | 使开发者、科学家和工程师能够轻松高效地运行数千个批处理作业。 |
| 实例 (虚拟机) | Elastic Container Registry (ECR) | 轻松存储、管理和部署容器镜像。                               |
| 实例 (虚拟机) | Elastic Kubernetes Service (EKS) | 全托管的 Kubernetes 服务。                                   |
| 实例 (虚拟机) | Fargate                          | 无服务器计算，用于容器。                                     |
| 实例 (虚拟机) | Lambda                           | 在无需考虑服务器的情况下运行代码。仅为使用的计算时间付费。   |
| 实例 (虚拟机) | Wavelength                       | 为 5G 设备提供超低延迟应用程序。                             |
| 实例 (虚拟机) | Cloud on AWS                     | 以更快的速度进行创新，快速迁移到云，并从任何位置安全工作。   |
| 实例 (虚拟机) | Local Zones                      | 将延迟敏感型应用程序运行得更靠近最终用户。                   |

**存储**

| 服务            | 描述                                                         |
| --------------- | ------------------------------------------------------------ |
| S3              | 互联网的存储库，即构建用来存储和检索任何数量数据的对象存储。 |
| EBS             | Amazon Elastic Block Store 是一种提供块级存储卷的 Web 服务。 |
| EFS             | 为用户的 Amazon EC2 实例提供文件存储。                       |
| FSx             | 提供完全托管的第三方文件系统，具有原生兼容性和特性集。       |
| Storage Gateway | 将本地软件设备连接到云存储。                                 |
| DataSync        | 简化和快速地将大量数据在线迁移到本地存储和 S3、EFS 或 FSx for Windows File Server 之间。 |
| Transfer Family | 提供完全托管的，支持直接进出 S3 的文件传输。                 |
| Snow Family     | 高度安全的，便携式设备，用于收集和处理边缘数据，并将数据迁移到 AWS 或从 AWS 迁移出去。 |

| --------------- | ------------------------------------------------------------ |

| Connect         | 为本地网络和 AWS 网络提供专用连接。                          |

**推荐的安全最佳实践**

* 启用 Redshift 审计日志记录。
* 启用虚拟私有云 (VPC) 流量日志记录。
* 为删除 CloudTrail 存储桶要求多因素身份验证 (MFA)。
* 在所有 AWS 上启用 CloudTrail 日志记录。
* 为 IAM 用户启用多因素身份验证。
* 为多模式访问启用 IAM 用户。
* 将 IAM 策略附加到组或角色。
* 定期轮换 IAM 访问密钥，并标准化选定的天数。
* 设置严格的密码策略。
* 将密码过期时间设置为 90 天，并防止重复使用。
* 不要使用标准标头强制实施客户可视化页面。
* 不要使用过期的 SSL/TLS 证书。
* 为 CloudFront 分发使用 HTTPS。
* 限制对 CloudTrail 存储桶的访问。
* 在静止状态下加密 CloudTrail 日志文件。
* 加密 Elastic Block Store (EBS) 数据库。
* 使用 IAM 角色配置对资源的访问。
* 确保 EC2 安全组没有打开大型端口范围。
* 配置 EC2 安全组以限制传入的 EC2 访问。
* 避免使用 root 用户账户。
* 在客户端和 ELB 之间连接时，使用安全的 SSL 密码。
* 在客户端和 ELB 之间连接时，使用安全的 SSL 版本。
* 使用标准命名 (标记) 约定来命名 EC2。
* 对 RDS 进行加密。
* 确保没有使用 root 账户访问密钥。
* 使用安全的 CloudFront SSL 版本。
* 在所有 Redshift 集群中启用 require_ssl 参数。
* 定期轮换 SSH 密钥。
* 最小化独立安全组的数量。
* 减少 IAM 组的数量。
* 终止未使用的访问密钥。
* 禁用对不活动或未使用的 IAM 用户的访问。
* 删除未使用的 IAM 访问密钥。
* 删除未使用的 SSH 公钥。
* 限制对 AMIS 的访问。
* 限制对 EC2 安全组的访问。
* 限制对 RDS 实例的访问。
* 限制对 Redshift 集群的访问。
* 限制出站访问。
* 在不常见的端口上禁止不受限制的入站访问。
* 限制对众所周知的端口的访问，例如 CIFS、FTP、ICMP、SMTP、SSH、远程桌面。
* 按数据存储类型、合规性要求和可能存在的威胁对所有现有自定义应用程序进行分类。
* 在整个开发过程中，都要让 IT 安全团队参与进来。
* 授予应用程序用户尽可能少的权限。
* 对所有自定义应用程序和所有其他云服务实施一套数据丢失预防策略。
* 对高度敏感数据（如受保护的健康信息 (PHI) 或个人身份信息 (PII)）进行加密。
